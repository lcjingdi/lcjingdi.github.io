利用代理服务器解决vpn下内网无法访问的问题

By lcjingdi

发表于 2017-10-13

文章目录
  1. 1. 前言：
  2. 2. 步骤
  3. 3. 总结

前言：

由于经常需要去google查一些资料，买了一个翻墙的软件，但是出现的问题是连接了翻墙软件后，再运行svn或者内网环境的开发，都会出问题，需要不停的切换内外网环境，造成了不必要的时间浪费，今天正好配置代理服务器的过程中解决了这个问题，记录一下

步骤

  1. 首先在电脑上打开vpn，能正常科学的上网
  2. 打开charles进行设置，具体设置如下
    • macOS Proxy勾选
    • 设置Proxy Settings…
      • Proxies
        • Port:8888
      • macOS
        • 勾选Enable macOS proxy
        • 勾选Use HTTP proxy
        • 勾选Enable macOS proxy launch
    • 设置SSL Proxying Settings…
      • SSL Proxying
        • 勾选Enable SSL Proxying
        • 在下面add进去需要使用SSL代理的IP地址（此处的操作主要是为了拦截一些ip的代理，如果不选择，则没有影响，因为在host中已经设置了访问某些域名的时候自动转为访问内网的ip）

总结

charles的作用主要是来充当代理服务器，局域网内任何设备只要把代理设置为此设备的ip和端口，则可以享受此ip提供的各种服务

例如在此电脑上开了

  1. ftp服务
  2. apache 的http服务
  3. http代理服务
  4. 远程登录服务

为什么说解决了翻墙和内网开发的问题呢？

  1. svn和翻墙的影响

    因为svn协议和http/https协议的不同，所以这两个没有影响，因为在两台电脑上

  2. 可以无证书安装的进行内网线下测试（安装证书只是为了截取请求）

charlse中->Proxy->SSL Proxying Settings

此处要注意

如果勾选Enable SSL Proxying的话,并且一般设置为 *.*的话会出现的问题是，所有的https都会走代理，但是如果不安装证书或者证书不被信任（iOS10以上安装完需要去设置信任）的话，会出现证书问题
