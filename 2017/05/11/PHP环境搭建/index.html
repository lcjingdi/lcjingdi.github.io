    配置系统介绍

    主配置文件：httpd.conf（httpd.conf:apache主配置文件，几乎可以完成apache所有配置）和（Extra/httpd-vhosts.conf:由主配置文件内部载入的扩展配置文件，最终会成为一个部分）

    分布式配置文件：分散到不同目录中的apache配置文件便于针对某个目录进行配置,.htaccess这个文件

    如何使用分布式配置文件？

    1. 主目录配置文件允许某个目录可以使用分布式配置文件

    2. 利用配置目录指令（在conf/httpd.conf或则会conf/extra/httpd-vhosts.conf中）

      Allowoverride All 允许分布式配置文件重写主配置的值

    安装MySQL

    是安装了一台MySQL数据服务器，基于C/S方式管理的

    C：Client 操作服务器必须使用相应的客户端（特定服务器的特定软件）

    S：Server

    位置：/usr/local/mysql

    建立与PHP的连接

    1. 在php.ini文件中打开php_mysql.dll

    2. 配置mysql目录

      extension_dir=”目录路径”

    配置虚拟主机

    为什么要配置虚拟主机？

    基于域名的虚拟主机

    配置目录：httpd.conf

    方法一：直接在文件的最下面配置

    方法二：

    httpd.conf里面有一个# Virtual hosts 下面的：

    Include /private/etc/apache2/extra/httpd-vhosts.conf

    虚拟主机的配置需要写在这个文件里面，在这个文件完成后再载入apache里面

    1. 如果是配置基于域名的就要配置

      NameVirtualHost *:80

      NameVirtualHost：表示开启一个基于域名的虚拟主机

      *:80 表示监听所有ip的80端口的虚拟主机的请求

    2. 配置虚拟主机

      ServerName:主机名

      DocumentRoot:主机地址（路径目录）

      格式：

    3. 通过域名解析，完成配置

      127.0.0.1 相应域名

    注意

    1. 默认主机问题

      因为是基于域名的虚拟主机，如果不能通过域名确定虚拟主机，则会访问默认主机（第一个虚拟主机）

    2. 目录权限问题

      apache可以设定目录的访问权限，通过apache的来完成目录的配置(参考httpd.conf)

      如果没有对相应的虚拟主机的目录进行权限配置，则使用默认值

      一般把目录权限写在配置虚拟主机的地方
